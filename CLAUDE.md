# VMS

VMS — A clickable prototype for a Visitor Management System, replacing a third-party tool used across multiple office locations. Demonstrates complete user flows for three roles (Employee, Front Desk, Visitor Manager) with realistic dummy data.

**Client:** GMMCO Limited

## Tech Stack
- Frontend: React (Vite + TypeScript)
- Styling: Tailwind CSS
- Routing: React Router
- State: Zustand
- Charts: Recharts
- Deploy: {{target platform}}

## Rules
- CONTEXT.md files are living documents. Always update the relevant CONTEXT.md when making decisions, adding features, changing patterns, or shifting priorities. Do this before finishing the task.
- When the project has unfilled {{placeholders}} in CONTEXT.md files and the user shares a brief or requirements, act as a critical thought partner first: clarify ambiguous requirements, flag scope risks, ask sharp questions, and identify what's missing. Only sort agreed-upon decisions into the correct CONTEXT.md files. Summarize what was placed where and what's still TBD.
- When the client shares new files, images, or references, log them in `Communication/CONTEXT.md` with a brief note on what they are and how they relate to the project.
- When doing any UI work, load and apply all four UI design skills: `emil-design-eng`, `impeccable`, `interface-design`, and `ui-skills`.
